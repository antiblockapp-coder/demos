# Инструкция по настройке и обновлению GitHub Pages

## Первоначальная настройка

### 1. Создайте репозиторий на GitHub
- Перейдите на https://github.com/new
- Создайте новый репозиторий (например, `website` или `your-username.github.io`)
- **НЕ** инициализируйте с README, .gitignore или лицензией (они уже есть)

### 2. Подключите удаленный репозиторий
```bash
git remote add origin https://github.com/ВАШ-USERNAME/НАЗВАНИЕ-РЕПОЗИТОРИЯ.git
git push -u origin main
```

### 3. Настройте GitHub Pages
- Откройте репозиторий на GitHub
- Перейдите в **Settings** → **Pages**
- В разделе **Source** выберите:
  - **Branch**: `main`
  - **Folder**: `/ (root)` или `/web` (в зависимости от того, где находятся файлы сайта)
- Нажмите **Save**

## Обновление файлов на GitHub Pages

После внесения изменений в файлы выполните следующие команды:

```bash
# 1. Посмотреть, какие файлы изменены
git status

# 2. Добавить все изменения
git add .

# 3. Создать коммит с описанием изменений
git commit -m "Описание ваших изменений"

# 4. Отправить изменения на GitHub
git push
```

GitHub Pages автоматически обновит сайт в течение нескольких минут (обычно 1-5 минут).

## Важно: Расположение файлов сайта

**Вариант 1: Файлы в папке `/web`**
Если ваши HTML файлы находятся в папке `web/`, GitHub Pages нужно настроить на эту папку. Однако GitHub Pages не поддерживает подпапки напрямую. Рекомендуется:

1. Скопировать содержимое папки `web/` в корень репозитория
2. Или использовать GitHub Actions для автоматического деплоя из папки `web/`

**Вариант 2: Файлы в корне**
Если файлы сайта находятся в корне репозитория, просто выберите `/ (root)` в настройках GitHub Pages.

## Проверка сайта

После настройки ваш сайт будет доступен по адресу:
- `https://ВАШ-USERNAME.github.io/НАЗВАНИЕ-РЕПОЗИТОРИЯ/`
- Или `https://ВАШ-USERNAME.github.io/` (если репозиторий называется `username.github.io`)

## Быстрые команды для обновления

Создайте алиасы для удобства:

```bash
# Добавить в ~/.zshrc или ~/.bashrc
alias gpush="git add . && git commit -m 'Update website' && git push"
```

Тогда можно просто выполнить `gpush` для обновления сайта.
